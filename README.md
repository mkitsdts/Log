C++低性能单头文件跨平台日志库

看测试用例能知道，这个头文件只能达到百千级别的日志输出，鉴定为玩具无疑

日志分为四个等级，分别是：
* DEBUG
* INFO
* WARNING
* ERROR

已经实现:
* 日志内容包含等级和输出信息
* 多线程异步输出日志
* 日志内容时间

未来实现：
* 数据恢复

提升程序性能的方法：
* 修改日志时间生成的方式

保证线程安全的方法：
* 在消息队列达到上限时上锁

已知bug：主线程直接退出会导致日志无法完全输出（一定程度上已修复）。

还有个问题同时输出过多日志会导致内容缺失，目前不确定是什么问题造成。我觉得是插入队列的时候以及切换输出的时候间隙
